# 🧠 Study Guide: “LeetCode Patterns — Progress-Based System”

---

# 1. Core Idea (Very Important)

Instead of saying:

- “Finish this in 2 weeks”

You will say:

> “I don’t move forward until I reach mastery criteria.”

So every pattern becomes:

- Section → Lessons → Completion Criteria

---

# 2. Course Structure (Like a Real Curriculum)

---

## 🟢 SECTION 1 — Arrays & Hashing

### 🎯 Goal

Understand how to trade space for speed using hash-based lookups.

---

### 📚 Lessons

#### Lesson 1 — Frequency Maps

- Counting occurrences
- Using hashmap for O(1) lookup

#### Lesson 2 — Prefix Sum

- Precomputing cumulative values
- Range queries optimization

---

### ✅ Completion Criteria

You can move forward only if:

- You can **recognize this pattern in <30 seconds**
- You can solve:
  - Two Sum
  - Valid Anagram
  - Subarray Sum Equals K
- You can explain:
  > “Why hashmap reduces time complexity”
